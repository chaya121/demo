# Apparel Creations - ใบดีขั้นตอนผลิต (React + Vite Refactoring)

โปรเจกต์ **Apparel Creations** เป็น Web Application แบบ Offline-First ที่ใช้สำหรับสร้างและจัดการ "ใบดีขั้นตอนผลิต" สำหรับโรงงานผลิตเสื้อผ้า โดยผู้ใช้งานสามารถบันทึกรายละเอียดการผลิต ขั้นตอนการเย็บ การประเมินราคา และติดตามผลการผลิตจริงได้

## 🛠️ รายละเอียดการปรับปรุงระบบ (Refactoring Journey)

ระบบเดิมเป็นโครงสร้าง **HTML/CSS/JS** ทั้งหมดรวมอยู่ในไฟล์เดียว (`index.html`) จำนวนเกือบ 1,000 บรรทัด ซึ่งทำให้การบำรุงรักษา (Maintenance) จัดการได้ยาก จึงได้ทำการรื้อโครงสร้างใหม่ (Refactoring) ทั้งหมดให้อยู่ในรูปแบบของ **React 18 + Vite** โดยมีรายละเอียดสิ่งที่ทำไปทั้งหมดดังนี้:

1. **Scaffold Project**: สร้างโปรเจกต์ใหม่ผ่าน Vite โดยใช้เทมเพลต React
2. **Component-Based Architecture**: ซอยโค้ดเดิมที่รวมกันในไฟล์เดียว ออกมาเป็น UI Component ย่อยๆ เพื่อให้นำกลับมาใช้ซ้ำได้ง่าย
3. **State Management**: เปลี่ยนการดึงค่าจาก DOM (เช่น `document.getElementById`) มาเป็นการใช้ React State (`useState`) เป็น Source of Truth
4. **Data Persistence**: เขียน Custom Hook (`useLocalStorage`) เพื่อผูก React State เข้ากับ `localStorage` ทำให้ข้อมูลถูกเซฟลงเครื่องอัตโนมัติ ไม่หายเมื่อรีเฟรช
5. **CSS Modularity**: ย้ายโค้ด CSS เดิมทั้งหมดมาจัดระเบียบใหม่ในไฟล์ `global.css` โดยใช้ CSS Custom Properties เพื่อให้ง่ายต่อการปรับเปลี่ยน Theme ในอนาคต
6. **Documentation**: จัดทำเอกสารแนะนำโฟลเดอร์ต่างๆ (`START_HERE.md`, `GETTING_STARTED.md`, ฯลฯ) เพื่อให้ Developer คนถัดไปสามารถมาอ่านแล้วทำงานต่อได้ทันที
7. **Token Sequence Builder (ขั้นตอนการเย็บแบบต่อประโยค)**: ปรับปรุงส่วน "ขั้นตอนการเย็บ" จากการกด Toggle ปุ่มสี ให้กลายเป็นระบบ **สร้างประโยคแบบ Token** — ผู้ใช้สามารถพิมพ์ข้อความได้อิสระ และกดปุ่มคำศัพท์ (เย็บติด, ปกเสื้อ ฯลฯ) เพื่อแทรกเป็น "Tag สี" ลงในช่องกลางประโยคได้ตรงตำแหน่ง โดย Tag แต่ละชนิดจะแสดงสีต่างกันตามหมวดหมู่ (การกระทำ=น้ำเงิน, ชิ้นส่วน=เขียว, ทิศทาง=เหลือง, ขนาด=ชมพู)

## 📂 โครงสร้างและหน้าที่ของแต่ละไฟล์ (File Structure & Roles)

เพื่อให้ง่ายต่อการศึกษาโค้ด ด้านล่างคือสรุปหน้าที่ของทุกไฟล์ที่ถูกสร้างขึ้นในโปรเจกต์นี้:

### 1. ไฟล์แกนหลัก (Core Files)
- **`package.json`**: ไฟล์จัดการไลบรารีของโปรเจกต์ (Dependencies) กำหนดให้ใช้ React 18 และ Vite
- **`vite.config.js`**: ไฟล์ตั้งค่าของ Vite Server (รองรับ React Plugin)
- **`index.html`**: ไฟล์เริ่มต้นของหน้าเว็บ มีแค่แท็ก `<div id="root"></div>` เพื่อให้ React เข้ามาแทรกเนื้อหา
- **`src/main.jsx`**: จุดเริ่มต้นที่แท้จริงของ React มีหน้าที่นำ `App.jsx` ไป Render ใส่หน้า `index.html`
- **`src/App.jsx`**: ไฟล์ Component ศูนย์กลาง ทำหน้าที่เก็บ State หลักของระบบ (ข้อมูลใบดี และ หน้าแท็บปัจจุบัน) และควบคุมว่าจะแสดงหน้าจอไหน
- **`src/styles/global.css`**: ไฟล์รวมสไตล์ (CSS) และโทนสีหลัก (Variables) ทั้งหมดของระบบ

### 2. หน้าจอหลัก (Pages)
เก็บอยู่ที่โฟลเดอร์ **`src/pages/`**
- **`FormPage.jsx`**: หน้าฟอร์มหลักสำหรับการกรอกและสร้างใบดีใหม่ (แบ่งย่อยเป็นส่วนข้อมูลทั่วไป, ขั้นตอนการเย็บ, และผลิตจริง)
- **`DownloadPage.jsx`**: หน้าประวัติ/PDF ทำหน้าที่แสดงรายการใบดีที่เคยบันทึกไว้ สามารถกดลบข้อมูลได้
- **`StatsPage.jsx`**: หน้า Dashboard สรุปสถิติ นำข้อมูลการผลิตทั้งหมดมาคำนวณและแสดงผลรวม รวมถึงจัดกลุ่มสถิติตามแบรนด์เสื้อผ้า

### 3. ชิ้นส่วน UI (Components)
เก็บอยู่ที่โฟลเดอร์ **`src/components/`**
- **`Header.jsx`**: แถบโลโก้ Apparel Creations และชื่อระบบด้านบนสุด
- **`TabBar.jsx`**: แถบปุ่มนำทาง (Navigation) สำหรับกดสลับไปยังหน้า Form, Download, หรือ Stats
- **`Toast.jsx`**: ป๊อปอัปแจ้งเตือนมุมล่างของจอ (เช่น ข้อความ "บันทึกสำเร็จ") 
- **`MerChips.jsx`**: ปุ่มกดเลือกสีของผู้ดูแล (Mer) สามารถกดเลือกได้หลายสี
- **`StepBuilder.jsx`**: ระบบ **Token Sequence Builder** สำหรับสร้างลิสต์ขั้นตอนการเย็บแบบไดนามิก แต่ละขั้นตอนมีกล่องกลางสำหรับสร้างประโยค ผู้ใช้สามารถพิมพ์ข้อความอิสระ และกดปุ่มคำศัพท์ด้านล่าง (ประเภทการกระทำ, ชิ้นส่วน, ทิศทาง, ขนาด) เพื่อแทรก Tag สีลงกลางประโยคได้ทันที กด ✕ เพื่อลบ Tag ออกจากประโยคได้
- **`PreviewModal.jsx`**: หน้าต่างป๊อปอัปขนาดใหญ่ที่แสดงผลสรุปข้อมูลฟอร์มทั้งหมด ให้ผู้ใช้งานตรวจสอบก่อนกดยืนยันบันทึก

### 4. ฟังก์ชันช่วยเหลือ (Hooks & Utils)
- **`src/hooks/useLocalStorage.js`**: Custom Hook ของ React ทำหน้าที่ซิงค์ข้อมูลลง `localStorage` ทันทีที่มีการแก้ State 
- **`src/utils/storage.js`**: ฟังก์ชันช่วยเหลือสำหรับ CRUD (Create, Read, Update, Delete) โดยตรงกับตัวฐานข้อมูลจำลอง (คีย์ `acRec3`)
- **`src/constants/index.js`**: แหล่งรวบรวมตัวแปรคงที่ เช่น รายชื่อปุ่ม Tag เริ่มต้น (คำศัพท์ที่ใช้ในการเย็บ), และพาเลทสีต่างๆ

## 🚀 วิธีการรันโปรเจกต์เพื่อพัฒนา (Getting Started)

1. ตรวจสอบให้แน่ใจว่าติดตั้ง **Node.js** แล้ว
2. เปิด Terminal ในโฟลเดอร์โปรเจกต์
3. ติดตั้งไลบรารีทั้งหมด:
   ```bash
   npm install
   ```
4. สตาร์ท Local Server สำหรับรันหน้าเว็บ:
   ```bash
   npm run dev
   ```
5. เปิดเบราว์เซอร์แล้วไปที่ [http://localhost:5173](http://localhost:5173) เพื่อเริ่มใช้งาน
